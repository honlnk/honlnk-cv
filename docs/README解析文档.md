# README.md 简历解析系统文档

## 📖 概述

本项目实现了从README.md文件自动解析简历数据的系统，支持将结构化的Markdown内容转换为TypeScript数据对象，实现简历内容的自动更新。

## 🏗️ 系统架构

### 核心组件

- **`useResumeData.ts`**: 主要的composable，负责数据加载和解析
- **`parseResumeFromMarkdown()`**: 核心解析函数
- **错误处理机制**: 完善的错误处理和回退策略

### 数据流

```
README.md → fetch → parseResumeFromMarkdown → ResumeData → Vue组件渲染
```

## 🔍 解析规则详解

### 1. 标题解析

**规则**: 匹配一级标题，按 `|` 分割
```markdown
# 鸿影 | Java全栈开发工程师 | 21岁
```

**解析结果**:
- `name`: "鸿影"
- `position`: "Java全栈开发工程师"

### 2. 基本信息解析

#### 2.1 字段映射
| README字段 | 数据字段 | 解析规则 |
|-----------|----------|----------|
| `**姓名**: ` | `name` | 提取冒号后内容 |
| `**年龄**: ` | `position` | 追加到职位信息后 |
| `**职位**: ` | `position` | 提取冒号后内容 |
| `**期望城市**: ` | `contact.location` | 提取冒号后内容 |
| `**期望薪资**: ` | `contact.salary` | 提取冒号后内容 |

#### 2.2 联系方式解析
**规则**: 匹配图标前缀
```markdown
- 📱 19563491369  →  phone: "19563491369"
- ✉ email@example.com  →  email: "email@example.com"
- 🌐 https://example.com  →  website: "https://example.com"
- 🐱 https://gitee.com/user  →  gitee: "https://gitee.com/user"
```

### 3. 核心优势解析

**规则**: 三级标题作为分类，列表项作为具体内容
```markdown
### 全栈开发  ← 分类标题
- SpringBoot+Vue.js技术栈  ← 内容项1
- MySQL+Redis数据层      ← 内容项2
```

**数据结构**:
```typescript
{
  title: "全栈开发",
  items: ["SpringBoot+Vue.js技术栈", "MySQL+Redis数据层"]
}
```

### 4. 项目经历解析

#### 4.1 项目基本信息
**规则**: 从段落中提取角色和时间
```markdown
**角色**: 全栈开发 | **时间**: 2024.11-至今
```

**解析结果**:
- `role`: "全栈开发"
- `duration`: "2024.11-至今"

#### 4.2 项目亮点
**规则**: 列表项作为项目亮点
```markdown
- 集成牛津/百度翻译API实现多维度单词解析
- 创新应用IndexedDB实现浏览器端数据缓存
```

#### 4.3 技术栈
**规则**: 包含"**技术栈**:"的行或段落
```markdown
**技术栈**: Vue3+ElementPlus+SpringBoot3+Mybatis+Redis
```

**分隔符**: 支持多种分隔符
- `+` (加号)
- `、` (中文顿号)
- `,` (逗号)

### 5. 教育背景解析

#### 5.1 学校信息
**规则**: 从包含关键信息的段落中提取
```markdown
**学校**: 山东商业职业技术学院 | **专业**: 软件技术（专科） | **时间**: 2022-2025
```

**解析结果**:
- `school`: "山东商业职业技术学院"
- `major`: "软件技术（专科）"
- `duration`: "2022-2025"

#### 5.2 校园经历
**规则**: 列表项作为经历描述
```markdown
### 校园经历
- 自主学习15+项技术栈，独立开发2个项目
- 获得技能类比赛，国奖1项，省奖多项
```

### 6. 附加价值解析

**规则**: 三级标题中的图标作为标识，列表项作为内容
```markdown
### 🚀 技术视野  ← 图标+标题
- 持续跟进SpringAI/大模型应用开发
- 对通用大模型使用与训练有较高的兴趣
```

**支持的图标**:
- 🚀 (火箭) - 技术视野
- 🛡️ (盾牌) - 质量保障
- 📚 (书本) - 学习能力

**数据结构**:
```typescript
{
  icon: "🚀",
  title: "技术视野",
  content: ["持续跟进SpringAI/大模型应用开发", "对通用大模型使用与训练有较高的兴趣"]
}
```

## ⚠️ 注意事项

### 1. 格式严格性
- 章节标题必须使用 `##` 格式
- 子分类标题必须使用 `###` 格式
- 联系方式必须使用指定的图标前缀
- 技术栈行必须包含 `**技术栈**:` 关键字

### 2. 编码规范
- 使用UTF-8编码
- 避免特殊字符和表情符号（除指定图标外）
- 保持格式的一致性和缩进

### 3. 数据完整性
- 必要字段不应为空（姓名、职位、联系方式等）
- 项目和经历信息应保持真实性和准确性

## 🔧 开发指南

### 1. 修改README.md
直接编辑根目录下的README.md文件，修改后会自动在开发环境生效。

### 2. 添加新字段
如需添加新的解析字段，需要：
1. 在 `ResumeData` 接口中添加字段定义
2. 在 `parseResumeFromMarkdown()` 函数中添加解析逻辑
3. 更新对应的Vue组件以显示新字段

### 3. 调试解析问题
解析器会在控制台输出详细日志，包括：
- 解析进度
- 错误信息
- 警告信息

### 4. 错误处理
- 解析失败时自动回退到静态数据
- 网络错误时显示重试按钮
- 格式错误时给出具体的错误提示
